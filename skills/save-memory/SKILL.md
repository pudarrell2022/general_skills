# Update Agent

## 目标

将一次对话里已经确认、可长期复用的项目信息沉淀到项目 `AGENTS.md`，避免重复口头说明。

## 执行流程

1. 收集上下文。
读取项目现有 `AGENTS.md`，并按需读取 `README.md`、`README.zh-CN.md`、`Package.swift`、`Sources/` 目录结构，确认事实。

2. 提取可复用知识。
仅保留稳定且可操作的信息：项目目的、技术栈、目录职责、关键命令、数据/状态流、约定格式、构建与发布入口。

3. 更新而非重写。
优先在原有章节内增补；仅在缺失时新增小节。保持中文、简洁、可扫描，避免与现有内容重复。

4. 保持准确边界。
不写未经验证的推测；不写临时讨论结论；不覆盖用户已有的自定义规则。

5. 交付变更摘要。
输出本次新增/调整了哪些知识点，并给出修改文件路径。

## 写作约束

- 使用中文。
- 使用明确路径和命令（例如 `Sources/ViewModels/`、`swift build`）。
- 每条信息都应可直接指导后续 AI 助手工作。
- 若本次对话无新增有效知识，明确说明“无需更新”。
