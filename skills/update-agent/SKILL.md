# Update Agent

## 目标
生成可长期复用的项目级说明文档，保持简洁、可扫描、可执行，供后续 AI 助手和人类贡献者快速上手。

## 执行流程
1. 盘点仓库事实。
- 读取现有 `AGENTS.md`（若存在）。
- 扫描目录与关键文件：优先 `README*`、`pyproject.toml`/`package.json`/`Makefile`、构建脚本、测试配置、部署脚本。
- 用真实文件和命令确认信息，不写猜测。

2. 判定创建或更新。
- 若 `AGENTS.md` 不存在：按标准结构新建。
- 若 `AGENTS.md` 已存在：保留用户自定义规则，在原章节内补全或修正，避免整篇重写。

3. 组织核心内容（必须覆盖）。
- 说明项目在做什么：目标、输入输出、主要运行方式。
- 说明文件结构与职责：列出关键目录/文件及作用。
- 说明环境与开发约定：语言版本、依赖工具、常用命令、代码风格与命名规范。
- 补充必要信息：测试方式、提交流程、部署注意事项、安全/配置边界。

4. 产出可执行文本。
- 使用明确路径与命令示例（如 `uv run build.py`、`src/`、`tests/`）。
- 保持简练，避免空话；优先项目特定信息。
- 若存在缺失信息，明确标注“当前仓库未发现”，不要臆造。

5. 交付结果。
- 输出本次新增/更新的要点。
- 明确修改文件路径：`AGENTS.md`。

## 写作约束
- 使用中文。
- 使用 Markdown 标题分层，便于快速浏览。
- 优先 200-400 词（或相当篇幅）的精简指南；按仓库复杂度适当调整。
- 不记录一次性讨论内容，只沉淀稳定规则。
