# Auto Push

## 目标

复用 `$auto-commit` 完成规范提交，然后仅处理远端检查与 `git push`，避免重复维护提交规则。

## 执行流程

1. 调用提交技能
- 先调用 `$auto-commit`，直接复用其仓库检查、暂存策略、`.gitignore` 补全、提交信息生成与安全约束。
- 若 `$auto-commit` 执行失败，则停止流程并返回失败原因，不继续推送。

2. 检查远端
- 提交成功后，检查是否存在可用远端：`git remote -v`。
- 若没有远程仓库，明确告知用户当前仓库尚未配置 remote，本次流程到提交为止。
- 若存在远端但当前分支未设置上游，允许直接执行 `git push`；若 Git 要求用户改用带上游参数的命令，再把原始原因反馈给用户。

3. 执行推送
- 在已存在远端的前提下，运行 `git push` 推送当前分支到默认远端。
- 若推送失败，输出 Git 原因（如无上游分支、鉴权失败、非快进冲突）并给出下一步建议。

4. 回报结果
- 输出最新提交信息：`git log -1 --oneline`。
- 输出推送结果摘要：目标远端、分支名、是否成功。

## 质量与安全要求

- 不改写历史：除非用户明确要求，不执行 `--force`、`push --force-with-lease`、`rebase`、`reset --hard`。
- `auto-push` 不重复定义提交规则；提交相关行为统一以 `$auto-commit` 为准。
- 遇到冲突或权限问题时，先报告现状与可选处理方式，再等待用户决定。
- 全程保持简洁输出，重点说明“做了什么、结果如何、下一步是什么”。

## 快速命令模板

```bash
# 1) 先调用提交技能
$auto-commit

# 2) 检查远端
git remote -v

# 3) 再推送
git push

# 4) 查看最新提交
git log -1 --oneline
```
